# 使用指南（GUI）

下面基于 `yoloGui.py` 中的界面元素，逐项说明如何使用该 GUI 进行训练配置与启动。

1. 启动程序
   - 运行：`python yoloGui.py`。

2. YOLO 版本（`YOLO版本:`）
   - 可选：`YOLOv8`、`YOLOv11`、`YOLOv26`。
   - 选择不同版本会影响可用的模型类型（`模型类型` 下拉框会动态更新）。

3. 任务类型（`任务类型:`）
   - 三项：`检测 (detect)`、`分割 (segment)`、`分类 (classify)`。
   - 选择任务类型后，`模型类型` 会限制为对应任务的模型（例如 `-seg` 后缀表示分割模型）。

4. 模型类型（`模型类型:`）
   - 动态更新：根据上面的 YOLO 版本与任务类型显示候选模型，例如：
     - YOLOv8 检测：`yolov8n`、`yolov8s`、`yolov8m`、`yolov8l`、`yolov8x`。
     - YOLOv11/YOLOv26 类似，但前缀分别为 `yolov11*` 或 `yolov26*`。
   - 也有分割/分类变体（例如 `yolov11m-seg`、`yolov26s-cls`）。

5. 预训练权重（`使用预训练权重`）
   - 勾选则利用已有 `.pt` 权重文件（例如 `yolov8n.pt`），否则 GUI 会尝试使用 YAML 配置文件创建新模型（根据模型类型推断 YAML 名称）。

6. 输入尺寸（`输入尺寸:`）
   - 用于训练/推理的输入图像短边或长边（取决于模型实现），在 GUI 中以整数框输入（默认 640）。

7. 权重文件名（`权重文件名称:`）与保存路径
   - 指定训练完成后保存的权重文件名与目标目录。

8. 数据路径、类管理与其它设置
   - GUI 提供训练/验证/测试路径选择与清除按钮，以及类别编辑、加载与清空操作。

9. 启动训练/监控
   - 点击训练相关按钮（在界面顶部或指定位置）启动训练，训练由 `YOLOTrainingThread` 在后台运行并通过信号更新进度/日志。

10. 日志与错误
   - 程序会在界面中显示日志、警告与训练指标；如加载模型失败，会尝试降级加载默认轻量模型。

常见问题
- 如果选择 `YOLOv11` 后没有看到正确的模型类型，请确认 `yoloGui.py` 中 `model_family_combo` 文字为 `YOLOv11`（已包含修复）。

如需自动保存/加载上次配置或更多高级使用示例，请查看 `CONFIG.md`。
